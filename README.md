## What is Ascender Bare?

Ascender Bare is a [GitHub repository template](https://docs.github.com/en/repositories/creating-and-managing-repositories/creating-a-template-repository) designed for environments where Docker cannot be used (for example, [ABCI](https://abci.ai/en)). It aims to ensure reproducibility of Python-based research experiments as much as possible while maintaining efficiency. It is developed based on [Ascender](https://github.com/cvpaperchallenge/Ascender), an existing Docker-based project template. It incorporates several pre-implemented features to expedite development:

- **Virtual Environment / Toolchain Management / Task Runner**: The management of the Python runtime and core development tools is handled by [mise](https://mise.jdx.dev).
- **Virtual Environment / Python Package Management**: Development environment reproducibility ensured by [uv](https://docs.astral.sh/uv).
- **Coding Style**: Automatic code linting and formatting with [Ruff](https://docs.astral.sh/ruff/).
- **Static Type Checking**: Early bug detection assisted by [Mypy](https://github.com/python/mypy).
- **Testing**: Testing simplification achieved through [pytest](https://github.com/pytest-dev/pytest).
- **GitHub Integration**: Integration features including [GitHub Actions workflows](https://docs.github.com/en/actions/using-workflows/workflow-syntax-for-github-actions), issue templates, and more.

## Project Organization

The project is organized as follows, with each directory and file intended to serve the purposes described below.

```
    ├── .github/               <- GitHub-specific settings and workflows.
    │
    ├── data/                  <- Raw and processed datasets used in experiments.
    │
    ├── environments/          <- Configuration files for reproducible environments.
    │
    ├── models/                <- Trained, saved, or pretrained model artifacts.
    │
    ├── notebooks/             <- Jupyter notebooks.
    │
    ├── outputs/               <- Generated outputs such as logs, results, and visualizations.
    │
    ├── scripts/               <- Utility and helper scripts for project tasks.
    │
    ├── src/                   <- Main Python source code implementing the project.
    │
    ├── tests/                 <- Unit and integration test code.
    │
    ├── .dockerignore          <- Ignore rules for Docker builds.
    ├── .envrc                 <- Environmental variable loaded by direnv.
    ├── .gitignore             <- Ignore rules for Git version control.
    ├── mise.lock              <- Pins exact versions and checksums of tools for mise.   
    ├── mise.toml              <- Project tool/runtime configuration for mise.
    ├── LICENSE                <- Project license.
    ├── pyproject.toml         <- Python project metadata and dependency configuration.
    ├── README.md              <- Top-level project documentation for developers.
    └── uv.lock                <- Dependency lock file generated by uv.
```

## Prerequisites Installation

This section provides installation instructions. If you have already installed the prerequisites, you may skip this section.

### Install mise

Please install mise by following the instructions on [the official mise page](https://mise.jdx.dev/getting-started.html). You can install it using various package managers such as Homebrew or apt.

Or you can also install the binary using the following command.

```bash
# Please replace shell with the one you are using
% curl https://mise.run | bash
```

After installation, please add the following content to your shell configuration file (such as `.bashrc` or `.zshrc`)."

```bash
# Please replace shell with the one you are using
eval "$(mise activate bash)"
```

### Install direnv

Please install direnv by following the instructions on [the official direnv page](https://direnv.net/docs/installation.html). You can install it using various package managers such as Homebrew or apt.

Or you can also install the binary using the following command.

```bash
# Please replace shell with the one you are using
% curl -sfL https://direnv.net/install.sh | bash
```

After installation, please add the following content to your shell configuration file (such as `.bashrc` or `.zshrc`)."

```bash
# Please replace shell with the one you are using
eval "$(direnv export bash)"
```

> [!NOTE]
> In mise, the environment variable `MISE_DATA_DIR` is used to determine where plugins and tools are installed ([reference](https://mise.jdx.dev/directories.html#local-share-mise)). In Ascender Bare, we override `MISE_DATA_DIR` to point to the project’s root directory in order to consolidate dependency downloads there as much as possible. However, since the value of this environment variable is referenced at the moment mise starts up, mise itself cannot manage this value. At the same time, manually overriding the value every time would be inconvenient, so we use direnv to automatically override it instead.

> [!WARNING]
> Both mise and direnv have features for managing environment variables, and using them together can potentially cause conflicts. When developing on your own based on Ascender Bare, we recommend limiting the use of direnv to the bare minimum and relying on mise’s functionality for environment variable management.

## Quick Start

This section outlines how to begin using Ascender Bare.

### Create a GitHub Repository from Ascender Bare.

To start, you need to create your own GitHub repository from Ascender Bare:

- Visit the GitHub repository page of Ascender Bare.
- Click the ["Use this template"](https://github.com/new?template_name=AscenderBare&template_owner=cvpaperchallenge-alumni) button at the top right of the page.
- Complete the form and click the "Create repository" button.

Your new repository should now be set up in your GitHub account.

### Start Development

```bash
# Clone the repository
% git clone git@github.com:<YOUR_USER_NAME>/<YOUR_REPO_NAME>.git
% cd <YOUR_REPO_NAME>

# Allow direnv to load the .envrc file
% direnv allow .envrc

# Install the Python runtime and core development tools by mise
# All installations will be placed under .mise
% mise install -y 

# Install Python packages by uv
# All installations will be placed under .venv
% uv sync

# Running Python code in a uv environment built on top of the mise environment
% uv run python src/sample.py
```

You are now ready to start developing with Ascender Bare.

> [!NOTE]
> The important point here is that, except for mise itself, all dependencies—including the Python runtime—are consolidated under `.mise` and `.venv` in the project root. This enables us to efficiently build a highly reproducible environment without using Docker.

### Tasks

Use mise to run the common automation shortcuts defined in `mise.toml`. Refer to [the official mise documentation](https://mise.jdx.dev/tasks/) for more details.

```bash
# Show all available tasks and their descriptions.
% mise tasks
Name    Description                                                              
format  Run all formatting tasks (Ruff format/fix and mdformat)                  
lint    Run all lint checks (Ruff check, format check, mdformat check, and mypy) 
test    Run pytest with coverage reporting and slow-test durations  

# Apply formatting fixes across the codebase and documentation.
% mise format

# Run the lint suite (Ruff checks, mdformat check, and mypy).
% mise lint

# Execute the pytest suite with coverage reporting.
% mise test
```

### Rebuild the Environment

If the environment becomes corrupted, you can delete `.mise` and `.venv`, then reinstall to reconstruct the same environment like following.

```bash
% rm -rf .venv
% rm -rf .mise

% direnv allow .envrc
% mise install -y 
% uv sync
```

## Use Cases

### ABCI

[ABCI (AI Bridging Cloud Infrastructure)](https://abci.ai/en) is the world's first large-scale Open AI Computing Infrastructure, constructed by National Institute of Advanced Industrial Science and Technology (AIST).

ABCI does not support Docker. It is possible to use [Singularity](https://sylabs.io/docs/) instead, but there is a learning cost for Singularity. By using Ascender Bare, you can easily build a reproducible experimental environment. [Here you can find the documentation and job script templates for use with ABCI](./scripts/abci/abci.md).
